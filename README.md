# השעון המשפחתי — גרסה פשוטה למשפחות לא טכניות

המטרה של הגרסה הזו:
אנשים מבוגרים או לא טכניים נכנסים לאתר, לוחצים על יצירת לוח, ומקבלים שני קישורים ייחודיים:

1. קישור לטאבלט / מסך המטופל/ת
2. קישור ל-Backoffice / עריכה למשפחה

הם לא צריכים לדעת Google Sheets, Apps Script, Firebase או שום דבר טכני.

---

## איך זה נראה למשתמשים

המשפחה פותחת את האתר ורואה:

```txt
יצירת לוח משפחתי חדש
שם משפחה / שם המטופלת
[יצירת לוח וקבלת קישורים]
```

אחרי לחיצה הם מקבלים:

```txt
קישור לטאבלט:
https://.../?board=cohen-abc123...

קישור ל-Backoffice:
https://.../backoffice/?board=cohen-abc123...
```

זה כל מה שהם צריכים.

# מה את, מנהלת האתר, צריכה לעשות פעם אחת

## 1. ליצור Google Sheet

1. פתחי Google Drive.
2. צרי Google Sheet חדש.
3. תני לו שם, למשל:
   `השעון המשפחתי - נתונים`

## 2. להדביק Apps Script

1. בתוך הגוגל שיט לחצי:
   `Extensions` → `Apps Script`
2. מחקי את הקוד הקיים.
3. פתחי בקובץ ה-ZIP:
   `apps-script/Code.gs`
4. העתיקי את כל הקוד והדביקי ב-Apps Script.
5. לחצי Save.

## 3. לפרוס כ-Web App

1. ב-Apps Script:
   `Deploy` → `New deployment`
2. בחרי סוג:
   `Web app`
3. Execute as:
   `Me`
4. Who has access:
   `Anyone`
5. לחצי Deploy.
6. אשרי הרשאות.
7. העתיקי את ה-Web App URL.

הוא נראה בערך כך:

```txt
https://script.google.com/macros/s/AKfycbx.../exec
```

## 4. להדביק את ה-URL בשני הקבצים

פתחי:

```txt
index.html
backoffice/index.html
```

חפשי בשניהם:

```js
const OWNER_APPS_SCRIPT_URL="";
```

והחליפי ל:

```js
const OWNER_APPS_SCRIPT_URL="https://script.google.com/macros/s/AKfycbx.../exec";
```

## 5. להעלות ל-GitHub Pages

העלי לשורש הריפו:

```txt
index.html
.nojekyll
README.md
backoffice/
apps-script/
```

GitHub Pages:
- Source: Deploy from a branch
- Branch: main
- Folder: /root

# שימוש למשפחה

1. המשפחה נכנסת לאתר.
2. כותבת שם משפחה / שם המטופלת.
3. לוחצת:
   `יצירת לוח וקבלת קישורים`
4. שומרת את שני הקישורים.
5. פותחת את קישור הטאבלט בטאבלט של המטופל/ת.
6. פותחת את קישור ה-Backoffice במחשב שלה.

כל שינוי ב-Backoffice מתעדכן בטאבלט אוטומטית בדרך כלל תוך עד דקה.

זה הרבה יותר יציב לשימוש חינמי עם Google Apps Script, במיוחד אם כמה משפחות משתמשות במערכת במקביל.

# טיפ חשוב לטאבלט: הוסף למסך הבית

כדי שהלוח ירגיש כמו דף משפחתי ולא כמו אתר:

1. פותחים את קישור הטאבלט בטאבלט או באייפד.
2. לוחצים על תפריט השיתוף בדפדפן.
3. בוחרים:
   `הוסף למסך הבית` / `Add to Home Screen`
4. יווצר אייקון רגיל על המסך.
5. מעכשיו אפשר לפתוח את הלוח כמו דף משפחתי, בלי שורת כתובת מבלבלת.

# תיקון V2 — שדה שם משפחה לא מאפשר הקלדה

בגרסה זו תוקן מצב שבו מסך יצירת הלוח נבנה מחדש כל שנייה ולכן שדה ההקלדה איבד פוקוס.
התיקון שומר את מסך הפתיחה יציב וגם עוצר את הטיימרים בזמן שנמצאים במסך יצירת/כניסת לוח.

כדי לוודא שהגרסה החדשה עלתה:
1. פתחו את האתר.
2. לחצו Ctrl+U.
3. חפשו:
   INPUT_FIX_V2_ONBOARDING_STABLE

# גרסת קישור אחד

בגרסה הזו יש קישור אחד בלבד לכל משפחה:

```txt
https://YOUR-SITE/?board=family-xxxx
```

מאותו קישור אפשר לעבור בלחיצה אחת בין:
- לוח המטופל/ת
- Backoffice / ניהול

הקובץ `patient.html` מכיל את תצוגת המטופל/ת.
הקובץ `backoffice/index.html` מכיל את הניהול.
הקובץ `index.html` הוא מעטפת אחת שמציגה את שניהם לפי בחירה.

# מזג אוויר לפי עיר

נוספה אפשרות ב-Backoffice:
`הגדרות חכמות` → `מזג אוויר לפי עיר`

אפשר לכתוב עיר, למשל:
- תל אביב
- ירושלים
- חיפה
- London
- Paris

וללחוץ `עדכון מזג אוויר לפי עיר`.

המערכת משתמשת ב-Open-Meteo Geocoding כדי למצוא Latitude/Longitude ושומרת אותם בלוח.
מסך המטופל/ת יציג מזג אוויר לפי העיר שנבחרה.

# מזג אוויר לפי מיקום המטופל/ת

בגרסה זו נוספה אפשרות אוטומטית:
אם בפרופיל המטופל/ת כתוב יישוב/עיר, למשל:

```txt
רוחמה
```

אפשר ללחוץ ב-Backoffice:

```txt
עדכון מזג אוויר לפי עיר/יישוב המטופל/ת
```

או להפעיל בהגדרות:

```txt
מזג אוויר לפי עיר/יישוב המטופל/ת
```

המערכת תחפש את המיקום דרך Open-Meteo Geocoding, תשמור Latitude/Longitude, והלוח יציג את מזג האוויר לפי אותו יישוב.

אם מיקום בעברית לא נמצא, מומלץ לנסות באנגלית, למשל:

```txt
Ruhama Israel
```

# שדרוגים בגרסה זו

## PWA — דף משפחתי למסך הבית
נוספו:
- `manifest.json`
- `sw.js`
- אייקונים
- רישום Service Worker

כך אפשר להוסיף את האתר למסך הבית והוא ייפתח יותר כמו דף משפחתי.

## העלאת תמונות בלי קישורים
ב-Backoffice, אנשי קשר ומשפחה יכולים עכשיו לבחור תמונה מהמחשב/טלפון.
התמונה מוקטנת בדפדפן ונשמרת כ-Base64 בתוך נתוני הלוח.

## קול עברי טוב יותר
קריאת הטקסט מנסה לבחור קול עברי איכותי יותר מהמכשיר, אם קיים.

## מזג אוויר
אם Open-Meteo לא נטען או אין אינטרנט, רכיב מזג האוויר פשוט מוסתר במקום להציג טעינה/שגיאה.

# שדרוגי אמון למשפחות לא טכניות

## שפה מרגיעה
השם בממשק הוא עכשיו `השעון המשפחתי`.
המסר למשפחות: לא צריך לפתוח חשבון, לא צריך סיסמה, ולא צריך להוריד שום דבר. אם יודעים לפתוח קישור מוואטסאפ — יודעים להשתמש בזה.

## מצב פסיבי למסך המטופל/ת
ב-Backoffice → הגדרות חכמות יש אפשרות:
`מצב פסיבי למסך המטופל/ת`

כשהוא פעיל, נגיעה במסך המטופל/ת לא עושה פעולה מבלבלת. במקום זה מופיעה הודעה קצרה:
`הכול בסדר. אין צורך ללחוץ.`

## שליחה בוואטסאפ
נוסף כפתור:
`📲 שליחה בוואטסאפ`

הוא יוצר הודעה מוכנה וחמה למשפחה עם הקישור היחיד.

## הדרכת מסך הבית
בפתיחה הראשונה מופיעה הדרכה ויזואלית קצרה שמראה איך להוסיף למסך הבית, כדי שהדף ייפתח כמו שעון קיר נקי וברור.

## ריפוי עצמי
במסך המטופל/ת, אם הסנכרון נכשל כמה פעמים ברצף או שהעמוד היה קפוא הרבה זמן, הדף מרענן את עצמו בשקט.
